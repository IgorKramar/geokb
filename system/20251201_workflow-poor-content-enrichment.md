# Рабочий процесс наполнения бедных на контент заметок по географии

Описание процесса работы с заметками, содержащими мало контента (менее 5 строк реального контента).

## Основной принцип

При наполнении бедной заметки ИИ должен:
1. **Анализировать существующую базу знаний** — находить связанные заметки, контекст, упоминания
2. **Задавать вопросы пользователю** — максимум вопросов по теме заметки для получения недостающей информации
3. **Синтезировать контент** — на основе ответов пользователя и существующей информации писать обновленную заметку

## Пошаговый процесс

### 1. Выбор заметки для наполнения

Пользователь указывает заметку из `system/GEOGRAPHY_POOR_CONTENT.md` или ИИ предлагает заметку на основе:
- Приоритетов (часто упоминаемые, ключевые концепции)
- Связности (заметки с большим количеством ссылок)
- Категории (страны, города, признаки)

### 2. Анализ существующей информации

ИИ анализирует базу знаний:

#### 2.1. Чтение самой заметки
- Читаю текущее содержимое бедной заметки
- Определяю, что уже есть (даже если это 1-4 строки)
- Выявляю основные темы и концепции

#### 2.2. Поиск связанных заметок
- Ищу упоминания этой заметки в других файлах через `grep`
- Нахожу связанные концепции через семантический поиск
- Анализирую контекст использования в существующих заметках

#### 2.3. Анализ категории и типа
- Определяю категорию заметки (страна, город, признак)
- Ищу похожие заметки того же типа для понимания структуры
- Изучаю примеры хорошо наполненных заметок в этой категории

**Пример:**
Если работаю с заметкой `20250101_france.md`:
- Читаю саму заметку (1 строка: "Франция — страна в Западной Европе")
- Ищу упоминания "Франция" в других заметках
- Ищу другие заметки о странах для понимания структуры
- Анализирую заметки о признаках для контекста

### 3. Формирование вопросов

На основе анализа ИИ формирует список вопросов для пользователя:

#### 3.1. Типы вопросов

**Для стран:**
- Какие дорожные знаки характерны? (форма, цвет, особенности)
- Какая архитектура типична? (стили, материалы, особенности)
- Какая растительность характерна? (деревья, ландшафты)
- Какой климат? (визуальные признаки климата)
- Какие языки используются? (на вывесках, знаках)
- Какие особенности инфраструктуры? (типы дорог, столбы, ограждения)
- Региональные различия внутри страны?

**Для городов:**
- Где находится? (страна, регион)
- Как выглядит? (архитектура, планировка)
- Какие особенности? (уникальные признаки)
- Какие дорожные знаки используются?
- Какая растительность характерна?
- Какие языки на вывесках?

**Для признаков:**
- Что это за признак? (описание)
- Где встречается? (страны, регионы)
- Как выглядит? (визуальные характеристики)
- Какие варианты существуют? (региональные различия)
- Как использовать для определения местоположения?

#### 3.2. Принцип максимума вопросов

ИИ задает **максимум вопросов**, которые могут помочь наполнить заметку:
- Не ограничиваюсь минимальным набором
- Задаю уточняющие вопросы
- Спрашиваю о деталях, которые могут быть важны
- Предлагаю варианты, если что-то неясно

**Пример вопросов для `france.md`:**
1. Какие дорожные знаки характерны для Франции? (форма, цвет, особенности)
2. Какая архитектура типична? (стили, материалы)
3. Какая растительность характерна? (деревья, ландшафты)
4. Какой климат? (визуальные признаки)
5. Какие языки используются на вывесках?
6. Какие особенности инфраструктуры? (типы дорог, столбы)
7. Есть ли региональные различия внутри Франции?

### 4. Получение ответов от пользователя

Пользователь отвечает на вопросы. ИИ:
- Запоминает все ответы
- Уточняет непонятные моменты
- Может задавать дополнительные вопросы на основе ответов

### 5. Синтез контента

На основе ответов пользователя и существующей информации ИИ пишет обновленную заметку:

#### 5.1. Структура заметки

Заметка должна содержать:
- **Основная информация** — общее описание
- **Признаки для GeoGuessr** — конкретные признаки для определения местоположения
- **Географические особенности** — ландшафт, климат
- **Культурные особенности** — язык, архитектура
- **Связи** — естественные ссылки на связанные заметки внутри текста

#### 5.2. Использование существующей информации

ИИ использует:
- Информацию из связанных заметок
- Контекст из упоминаний в других файлах
- Структуру похожих заметок
- Стиль и тон существующих заметок

#### 5.3. Естественные ссылки

Все ссылки создаются:
- Внутри текста, естественным потоком
- На основе упоминаний связанных концепций
- С проверкой через MOC и маппинг ссылок
- В правильном формате `[текст](./filename.md)`

### 6. Обновление заметки

После написания контента:
1. Обновляю заметку новым содержимым
2. Обновляю статус заметки в YAML FrontMatter (если нужно)
3. Проверяю, что все ссылки работают
4. Обновляю MOC-заметки, если заметка была добавлена в навигацию
5. Запускаю `python kb.py all` из корня проекта для обновления маппинга ссылок и списка бедных заметок

### 7. Коммит изменений

После наполнения заметки:
1. Перехожу в корень проекта
2. Получаю актуальный timestamp
3. Добавляю изменения: `git add .` (в корне проекта)
4. Создаю коммит: `git commit -m '[YYYYMMDD] - Наполнена заметка о [название]'`

## Примеры работы

### Пример 1: Страна

**Бедная заметка:** `20250101_france.md` (1 строка)

**Анализ:**
- Найдено упоминание в MOC
- Найдены другие заметки о странах для понимания структуры
- Определена категория: страна

**Вопросы пользователю:**
1. Какие дорожные знаки характерны для Франции?
2. Какая архитектура типична?
3. Какая растительность характерна?
4. Какой климат?
5. Какие языки на вывесках?
6. Особенности инфраструктуры?

**После ответов:**
- Пишу обновленную заметку с описанием признаков
- Добавляю ссылки на связанные заметки (дорожные знаки, архитектура)
- Обновляю статус заметки
- Запускаю `python kb.py all`

### Пример 2: Признак

**Бедная заметка:** `20250101_road-signs-europe.md` (2 строки)

**Анализ:**
- Найдены упоминания в заметках о странах
- Найдены другие заметки о признаках для понимания структуры
- Определена категория: признак

**Вопросы пользователю:**
1. Какие типы дорожных знаков характерны для Европы?
2. Какие особенности формы и цвета?
3. Какие различия между странами?
4. Как использовать для определения местоположения?

**После ответов:**
- Пишу обновленную заметку с описанием признака
- Добавляю ссылки на связанные заметки (страны, где встречается)
- Запускаю `python kb.py all`

## Интеграция в рабочий процесс

### После каждой работы с базой знаний

После обработки входящей информации по географии:
1. Запускаю `python kb.py all` из корня проекта
2. Это обновляет:
   - Маппинг ссылок (`system/LINK_MAPPING.json`, `system/LINK_MAPPING.md`)
   - Список бедных заметок (`system/GEOGRAPHY_POOR_CONTENT.md`)
   - Список битых ссылок (`system/GEOGRAPHY_BROKEN_LINKS.md`)

### Периодическое наполнение

Можно выделять время на наполнение бедных заметок:
- Выбирать категорию (страны, города, признаки)
- Работать с несколькими заметками подряд
- Использовать этот процесс для каждой заметки

## Важные принципы

1. **Максимум вопросов** — не ограничиваюсь минимальным набором, задаю все вопросы, которые могут помочь
2. **Анализ базы знаний** — всегда анализирую существующую информацию перед вопросами
3. **Синтез информации** — объединяю ответы пользователя с существующей информацией
4. **Естественные ссылки** — все ссылки внутри текста, не в отдельном разделе
5. **Обновление навигации** — после наполнения запускаю `python kb.py all` для обновления всех мета-файлов
